# Hコード・デッキシステム UI/UX設計提案書

## 1. コンセプト

FF13の「パラダイムシフト」システムに倣い、プレイヤーが事前にHコードの組み合わせを「デッキ」として構築し、ゲームプレイ中にリアルタイムで切り替えることで、義体のAIと思考様式を変化させる。これにより、プレイヤーは戦術的な判断に集中できる。

---

## 2. 主要なUI画面とコンポーネント

### 2.1. Hコード・デッキ編成画面

ゲームの拠点（義体パブなど）やメニューからアクセスできる、デッキを管理するためのメイン画面。

*   **画面構成:**
    *   **左側: Hコード・チップ・インベントリ**
        *   プレイヤーが所有しているHコード・チップが一覧で表示される。
        *   各チップには「コード(例: 29)」「名称(例: 最善の選択)」「説明」が表示される。
        *   ソート・フィルタリング機能（「戦闘用」「接客用」などで絞り込み）を持つ。
    *   **右側: デッキ設定エリア**
        *   **デッキ・スロット (6つ):** プレイヤーが構築できるデッキのリスト。FF13と同様、6つのスロットを用意する。
        *   各デッキスロットには、現在設定されているHコード・チップと、プレイヤーが自由につけられる**デッキ名**（例：「ボス戦用全力攻撃」「VIP接客モード」）が表示される。
    *   **中央: 義体スロット**
        *   デッキを選択すると、中央に現在操作対象の義体が表示される。
        *   プレイヤーは左のインベントリからHコード・チップをドラッグ＆ドロップ、あるいは選択して、この義体スロットに「インストール」する。
        *   （将来的には複数の義体を同時に操作する場合、ここに複数の義体スロットが並ぶことも想定）

*   **テキストによる画面イメージ:**
    ```
    +-----------------------------------------------------------------+
    |                     H-CODE DECK CONFIGURATION                   |
    +---------------------------+---------------------------+---------+
    | H-CODE CHIP INVENTORY     | PROSTHETIC BODY SLOTS     | DECKS   |
    |---------------------------|---------------------------|---------|
    |                           |                           |         |
    | [ ] 580: 自爆覚悟の全力攻撃 |       +---------------+   | [DECK 1] ボス戦用 |
    | [x]  29: 最善の選択       |       |               |   | [DECK 2] VIP接客 |
    | [ ]  20: 最悪の選択       |       |  義体モデルA  |   | [DECK 3] 探索用 |
    | [ ] 235: レベルアップの試練 |       | +-----------+ |   | [DECK 4] (空)  |
    | [ ]   6: 調和             |       | | Chip: 29  | |   | [DECK 5] (空)  |
    | [ ]   ...                 |       | +-----------+ |   | [DECK 6] (空)  |
    |                           |       +---------------+   |         |
    |                           |                           |         |
    +---------------------------+---------------------------+---------+
    ```

### 2.2. ゲームプレイ中のHUD（ヘッドアップディスプレイ）

戦闘中や接客中に、プレイヤーがHコード・デッキを操作するためのインターフェース。

*   **常時表示コンポーネント:**
    *   画面の隅に、現在アクティブなデッキ名と、設定されているHコードが表示される。（例: `[DECK 1: ボス戦用] - H-Code: 580`）

*   **デッキ切り替えUI:**
    *   特定のボタン（例: L1ボタン）を押している間、ゲーム画面がスローモーションになり、画面中央に**デッキ選択ホイール**が表示される。
    *   プレイヤーはスティックで切り替えたいデッキを選択し、ボタンを離すと即座にそのデッキ（Hコード）が有効になる。
    *   この操作感は、まさに「パラダイムシフト」のように、迅速かつ直感的であることが重要。

## 3. プレイヤーの体験フロー

1.  **準備フェーズ（拠点にて）:**
    1.  プレイヤーは「Hコード・デッキ編成画面」を開く。
    2.  想定される状況（次のミッションのボス、パブに来店予定のVIP客など）に合わせて、6つのデッキスロットにそれぞれ最適なHコード・チップを割り当てる。
    3.  各デッキに分かりやすい名前を付けて保存する。

2.  **実行フェーズ（戦闘・接客中）:**
    1.  ミッション開始。デフォルトのデッキ（例: DECK 1）がアクティブになる。
    2.  義体は、設定されたHコードに基づいて自律的に行動する（プレイヤーは`mv`/`mo`コマンドで大まかな指示を出す）。
    3.  戦況が変化（例: ボスが第二形態に移行）。
    4.  プレイヤーはデッキ切り替えUIを呼び出し、例えば「ボス戦用全力攻撃」デッキ（H-Code: 580）に瞬時に切り替える。
    5.  義体の行動と思考様式が即座に変化し、新たな状況に対応する。

このUI/UX設計により、プレイヤーはHコードシステムの深さを享受しつつも、直感的な操作で戦略的なプレイに没頭できるようになります。
